# 商业游戏敌人察觉 UI 系统分析报告

## 一、六大经典游戏逐案分析

### 1. Metal Gear Solid V: The Phantom Pain

MGSV 使用**弧形察觉指示器**，这是潜行游戏中最经典的参考实现。

- **察觉阶梯**：四阶段模型
  - 白色弧形（未察觉）→ 白色弧形微张（模糊感知，敌人转头看）→ 黄色弧形快速填充（确认可疑，敌人走向玩家方向）→ 红色弧形满格 + **(!) 图标**（完全暴露，进入 Combat Alert）
- **世界空间指示器**：弧形围绕在敌人头顶上方，玩家在任何角度都能看到
- **Reflex Mode**：察觉满格后短暂慢动作，屏幕边缘出现蓝色波纹特效，给玩家一次反杀窗口
- **最后已知位置**：白色虚影标记在玩家被最后看到的位置，敌人会调查该点
- **音频分层**：察觉时播放标志性的 "!" 音效（Metal Gear 系列经典音效），不同阶段音调不同

**核心设计哲学**：弧形比直线条更具威胁感，填充速度直接对应敌人反应速度，给玩家明确的"还差多少毫秒我就会被发现"的预期。

### 2. Splinter Cell: Blacklist

使用**环形检测指示器**，嵌入在 HUD 和敌人身上。

- **察觉阶梯**：HUD 中央的绿色检测环 → 环开始填充（绿→黄）→ 环满格闪烁（红）→ "Hunted" 状态横幅
- **状态标签系统**：屏幕顶部显示当前状态 —— "Hidden" → "Suspicious" → "Hunted" → "Engaged" → "Evaded" → "Hidden"
- **LKP (Last Known Position)**：白色半透明剪影 + 轮廓标记，敌人会向该位置搜索
- **敌人头顶指示**：小三角/箭头标记敌人的最后朝向，帮助玩家规划绕行路线
- **视觉风格**：军事科技感，HUD 元素类似夜视仪/战术目镜叠加

**核心设计哲学**：让玩家始终知道"系统对我的状态判定是什么"，消除不确定性。

### 3. Hitman: World of Assassination

使用**头顶弧形察觉条** + **小地图标记**的双通道系统。

- **察觉阶梯**：NPC 头顶出现白色弧形 → 弧形被黄色填充（速度取决于 NPC 类型和玩家行为）→ 弧形变红满格 → NPC 走向玩家质问 → 枪战
- **执法者 (Enforcer)**：特定 NPC 头顶有白色圆点/星形标记，他们能识破伪装，察觉条填充速度极快
- **伪装机制耦合**：不同伪装下，不同 NPC 类型的察觉速度和阈值不同（如保安对清洁工伪装不敏感）
- **小地图**：NPC 以白色点表示，察觉时变黄，战斗时变红，箭头表示朝向

**核心设计哲学**：察觉不仅仅是"看没看到"，而是"你是谁 + 你在哪 + 你做了什么"的综合评估。

### 4. Horizon Zero Dawn

使用**头顶察觉图标 + 弧形填充**，视觉语言清晰。

- **察觉阶梯**：无标记（未察觉）→ 黄色问号 `?` 图标出现（可疑，弧形开始填充）→ 黄色感叹号 `!`（确认威胁，弧形满格）→ 红色战斗状态
- **察觉弧**：围绕在敌人头顶的 180° 弧形，从黄色到红色的渐变填充
- **视野锥可视化**：部分敌人（如 Watcher）在 Focus 模式下可看到其视野范围
- **草丛指示**：玩家在草丛中时，屏幕边缘出现淡蓝色掩体标记

**核心设计哲学**：用最少的 UI 传达最多的信息，图标语言直观（`?` = 不确定，`!` = 确认）。

### 5. Batman: Arkham Knight

使用**敌人头顶威胁指示器** + **Predator Mode 专用 HUD**。

- **察觉阶梯**：敌人头顶无标记 → 白色弧形出现（开始注意）→ 弧形填充 + 心跳声渐强 → 敌人喊话警告同伴
- **威胁类型图标**：普通敌人无标记，武装敌人有枪械图标，带电/装甲敌人有特殊图标
- **Predator Mode**：屏幕变为冷色调，敌人骨骼透视显示，心跳监测器在屏幕角落显示 BPM
- **倒吊/高处指示**：非视觉察觉——敌人会注意到同伴失踪，头顶出现 `?` 然后开始搜索

**核心设计哲学**：玩家是"掠食者"而非"猎物"，UI 服务的是"策划下一次攻击"而非"躲避"。

### 6. Alien: Isolation

**刻意选择不显示敌人状态 UI**，这是最重要的反例。

- **为什么没有**：设计团队认为直接显示 Alien 的状态会削弱恐惧感和不可预测性
- **替代方案 —— Motion Tracker**：手持设备，只显示移动物体的距离和方向（一个亮点 = 可能是 Alien，也可能是其他），使用时发出声音会被 Alien 听到
- **替代方案 —— 音频系统**：Alien 的脚步声、管道爬行声、嘶吼声提示位置和状态，玩家需要"听"而不是"看"
- **替代方案 —— 环境叙事**：天花板的震动、滴落的液体、通风口的蒸汽

**核心设计哲学**：不确定性本身就是恐怖的核心机制。UI 是用来限制信息的，不是提供信息的。

## 二、通用设计模式总结

### 2.1 察觉度阶梯模型

| 级别 | 名称 | 视觉表现 | 玩家可操作？ | 典型触发 |
|------|------|----------|------------|---------|
| L0 | 无感知 | 无 UI | 完全自由 | 玩家在视野外/黑暗中 |
| L1 | 模糊注意 | 小图标/弧形微张 | 自由行动 | 余光看到、远处声响 |
| L2 | 可疑/警觉 | `?` + 填充弧形 | 可逃脱 | 看到玩家轮廓/光线 |
| L3 | 确认威胁 | `!` + 满格弧形 | 反杀窗口 | 清晰看到玩家 |
| L4 | 战斗/追猎 | 红色满格 + 状态标签 | 战斗/逃跑 | 确认后持续追踪 |

**DeEnergized 当前映射**：L0=Patrol, L1+L2=Suspicious, L3+L4=Investigate

### 2.2 世界空间 vs 屏幕空间

| 维度 | 世界空间 (Above-head) | 屏幕空间 (HUD) |
|------|----------------------|----------------|
| **优势** | 直观、不需要视线转移 | 不受遮挡、信息密度高 |
| **劣势** | 可能被遮挡、多敌人时混乱 | 需要视觉搜索、增加认知负荷 |
| **适用** | 近距离少量敌人 | 全局状态、多敌人 |

**建议**：主用世界空间头顶指示器，辅以屏幕边缘的方向箭头（当敌人不在视野内时）。

### 2.3 图标语言标准

| 图标 | Unicode | 语义 | 触发条件 |
|------|---------|------|---------|
| `?` | U+FF1F ？ | 可疑/不确定 | 感知到异常但未确认 |
| `!` | U+FF01 ！ | 确认威胁/警报 | 清晰识别到玩家 |
| `👁` | U+1F441 | 正在观察/搜索 | 调查状态 |
| `~` | — | 最后已知位置 | 玩家消失的位置 |
| `▲` | U+25B2 | 方向指示 | 屏幕边缘敌人方向 |

### 2.4 颜色编码标准

```
白/灰 (#CCC)  →  黄/橙 (#FA0)  →  红 (#F00)
   未察觉          可疑/警觉          警报/战斗
```

渐进式填充：0-30% 白色 → 30-70% 黄色 → 70-100% 橙色 → 满格红色。

### 2.5 填充速度设计

- **缓入加速**：察觉条前半段填充慢（给玩家反应时间），后半段加速（制造紧迫感）
- **非线性映射**：`fillAmount = awareness^1.3 / 100^1.3` 使前半段更慢
- **衰减**：通常是填充速度的 0.5-1 倍，让玩家有喘息空间

### 2.6 音频分层

| 察觉阶段 | 音频提示 | 作用 |
|---------|---------|------|
| L0→L1 | 微弱的上升音效 | 提醒"有人注意到你了" |
| L1→L2 | 音调上升 + 心跳声 | 制造紧张 |
| L2→L3 | 尖锐音效 + 敌人语音 | "被发现了！" |
| L3→L4 | 战斗音乐切入 | 状态转换 |

## 三、针对 DeEnergized 的设计方案

### 核心原则

1. **信息清晰但不破坏沉浸感** —— 世界空间 UI，最小化设计
2. **事件驱动解耦** —— LightSensor 暴露事件，UI 层订阅
3. **渐进式紧张** —— 颜色、大小、透明度随察觉度平滑变化
4. **可配置** —— 所有视觉参数可在 Inspector 中调节

### 视觉状态机

```
┌──────────────────────────────────────────────────┐
│  Patrol (awareness=0)                            │
│  UI: 完全隐藏 (alpha=0)                          │
│  音效: 无                                        │
├──────────────────────────────────────────────────┤
│  → 检测到光线/接近 → Suspicious                  │
├──────────────────────────────────────────────────┤
│  Suspicious (awareness 0→100)                    │
│  UI: 淡入 → 察觉条填充(黄→橙→红)                 │
│  图标: "？" (黄色)                               │
│  音效: 上升的紧张音效                            │
├──────────────────────────────────────────────────┤
│  → awareness=100 → Investigate                   │
├──────────────────────────────────────────────────┤
│  Investigate                                     │
│  UI: 条满格(红色) + 脉冲动画                     │
│  图标: "！" (红色) + 缩放弹跳                    │
│  音效: 警报音效 + 敌人语音                       │
├──────────────────────────────────────────────────┤
│  → awareness<100 → Suspicious (条衰减)           │
│  → awareness=0 → Patrol (UI 延迟淡出)           │
└──────────────────────────────────────────────────┘
```

### 实现架构

```
LightSensor (modified)
  + OnStateChanged(State old, State new)
  + OnAwarenessChanged(float value)
       │
       ▼
EnemyAlertUI (new)
  └── Canvas (World Space)
       ├── Slider (察觉条)
       │    ├── Background (暗色底)
       │    └── Fill Area → Fill Image (颜色动态)
       └── TextMeshProUGUI (状态图标)
            └── 显示 "？" / "！" / ""
```
